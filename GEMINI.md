# Project: Good Shepherd 2025 Theme

## Overview
Custom WordPress theme development environment using Vite and Tailwind CSS.

## Tech Stack
- **CMS:** WordPress
- **Bundler:** Vite (v7)
- **CSS:** Tailwind CSS (v4)
- **JS:** Vanilla ES Modules & jQuery

## Architecture & Workflow
- **Source:** `src/` (Edit files here)
    - `src/main.js`: Main entry point.
    - `src/style.css`: Main stylesheet (Tailwind imports).
- **Output:** `assets/` (Generated by build - DO NOT EDIT DIRECTLY)
    - `assets/js/`: Compiled JavaScript.
    - `assets/css/`: Compiled CSS.
- **Local Dev:** `https://goodshep2025.local` (Proxied via BrowserSync).

## Development Rules
1. **Vite Config:** Build output is customized to `assets/` (not `dist/`).
2. **Styling:** Use Tailwind utility classes whenever possible.
3. **PHP:** Ensure all PHP files are compatible with the latest WordPress version.
4. **Versioning:** Always update the `Version:` header in `style.css` when adding a new entry to `CHANGELOG.md`.
5. **Git Workflow:** Push changes to the repository immediately after completing a task or significant sub-task.

## Coding Standards & Patterns

### 1. Tailwind Colors
Use the custom theme classes defined in `style.css` instead of standard Tailwind palettes:
- `bg-purple` / `text-purple` (Primary Brand Color)
- `bg-red` / `text-red` (Secondary Brand Color)
- `bg-off-black` / `text-off-black` (Dark backgrounds)
- `bg-light-grey`, `bg-off-white`, `bg-light-purple` (UI backgrounds)

### 2. Helper Functions
- **Icons:** Always use `goodshep_icon( ['icon' => 'name', 'group' => 'utility'] )` instead of inline SVGs or `img` tags.
- **Block Wrappers:** Use `goodshep_get_block_classes()` to handle standard background colors and padding.
- **Debugging:** Use `preint_r( $var )` to print arrays/objects (visible only to admins).

### 3. AJAX Architecture
- **Handlers:** Place all AJAX actions in `inc/ajax-handlers.php`.
- **DRY:** AJAX handlers should return HTML generated by reusing existing template parts from `template-parts/components/` (e.g., `client_stories_item.php`), rather than rewriting HTML strings.

## Commands
- `npm run dev`: Start Vite dev server with BrowserSync proxy.
- `npm run build`: Compile assets for production.
